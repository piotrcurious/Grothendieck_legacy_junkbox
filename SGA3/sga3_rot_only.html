what the page is doing
Plain-language guide to the operations behind the picture and the code.

one idea

The circle does not care which full turn you add. 75°, 435°, and -285° all point to the same place on the ring. The code keeps one number for the running motion, then computes the wrapped angle with mod 360 before drawing the arrow.

how this matches SGA3

In Exposé VIII, a diagonalizable group is built from an ordinary commutative group M by the recipe D(M). In Exposé IX, a group of multiplicative type is one that becomes diagonalizable after a suitable faithfully flat quasi-compact cover. This demo only shows the simplest local behavior: the same object is written in a standard form after choosing the right view.

the three operations

add
Increase the raw angle by a chosen amount.
remove full turns
Subtract or add 360° until the angle sits in the usual range.
draw the same point
Use the wrapped value to place the arrow on the circle.

terms shown on screen

raw angle
The number that keeps growing while the motion continues.
wrapped angle
The angle after full turns are removed, kept inside one circle turn.
full turns
How many times 360° has been passed.
local view
The standard way of writing the same object after choosing a cover.
