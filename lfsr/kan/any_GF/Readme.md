# GF(p^n) LFSR Suite: Kan Extension Edition

This suite provides a robust, mathematically-grounded implementation of Linear Feedback Shift Registers (LFSRs) over arbitrary finite fields $GF(p^n)$. It utilizes a categorical framework based on **Kan Extensions** to bridge the gap between abstract Galois field theory and concrete software realizations.

## 1. Categorical Architecture

The system is structured as a mapping between a category of algebraic presentations and a category of implementation fragments.

### The Global Object
The "Global Object" in this suite is a point in the **Variety of Recurrences**. Specifically, it is defined by:
- A prime characteristic $p$ (defining the base field $\mathbb{F}_p$).
- An extension degree $n$.
- A monic irreducible polynomial $f(x) \in \mathbb{F}_p[x]$ of degree $n$.
- A primitive element $\alpha \in GF(p^n)^*$ (typically $\alpha = x \pmod{f(x)}$) which serves as the generator of the multiplicative group.

### Left Kan Extension (Lan) — Realization
The **Left Kan Extension** $(\text{Lan}_i F)$ takes the abstract global object and "extends" it into a set of local **Charts**. Each chart is a specific projection or coordinate system of the global orbit.
- **Companion Chart**: The standard state-space representation using the polynomial basis $\{1, x, \dots, x^{n-1}\}$.
- **Matrix Chart**: A linear-algebraic realization where the state transition is an $n \times n$ matrix $M$ over $\mathbb{F}_p$.
- **Trace Chart**: A projection $\text{Tr}: GF(p^n) \to \mathbb{F}_p$, reduced modulo 2 to produce a bit-stream. This represents the "shadow" of the high-dimensional orbit on the base field.
- **Decimation Chart**: A view of the orbit sampled at regular intervals (e.g., every 3rd step), corresponding to a different generator $\alpha^k$.
- **Reciprocal Chart**: The "dual" orbit generated by the reciprocal polynomial $f^*(x)$, effectively running the recurrence in reversed coordinates.

### Right Kan Extension (Ran) — Inference
The **Right Kan Extension** $(\text{Ran}_j G)$ implements the inverse process: reconstructing the global object from local observations.
- **State-level Inference**: If raw states are observed (Companion/Matrix), the generator is recovered via $\alpha = s_{i+1} / s_i$ in the field.
- **Linear-Algebraic Inference**: For Trace observations over $GF(2^n)$, the system solves a linear system $Ax = b$ where $A$ is the matrix of trace-projections of the basis elements. This allows efficient recovery of the initial state for large $n$.
- **Locus-Search Inference**: For complex projections where linearity is lost or the generator is unknown, the system searches the **Primitive Locus** — the set of all primitive elements in the field — to find a consistent global match.

## 2. Mathematical Detail

### NTL Context Management
Operating over arbitrary $GF(p^n)$ requires a two-level modular context in NTL:
1. `ZZ_pContext`: Manages the prime characteristic $p$.
2. `ZZ_pEContext`: Manages the extension modulus $f(x)$.

The `FieldContext` class encapsulates both, ensuring that `activate()` restores the exact algebraic environment required for arithmetic. This is critical when comparing objects from different fields (e.g., in inference) or different charts.

### Bit-Perfect Consistency
The **Categorical Gluing Test** verifies that the diagram commutes: specifically, that the Companion chart and Matrix chart, despite having different internal data structures and transition logic, produce bit-perfectly identical projections of the same global orbit.

## 3. Usage

### Installation
The suite requires NTL and GMP:
```bash
sudo apt-get update
sudo apt-get install -y libntl-dev libgmp-dev
```

### Compilation
```bash
g++ -O2 -std=c++17 gfq_lfsr.cpp -lntl -lgmp -pthread -o gfq_lfsr
```

### Running
Executing `./gfq_lfsr` runs the integrated test suite, demonstrating:
1. **Lan Realizations** across multiple primes ($p=2, 3, 5, 97$).
2. **Ran Inferences** from various chart observations.
3. **Consistency Checks** ensuring the mathematical integrity of the gluing process.
